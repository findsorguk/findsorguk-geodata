   SECTION ONE - HIGH LEVEL STRUCTURE


   The directory structure of the supplied data is shown below:

   

   ROOT
   |--README.txt
   |
   |--DOC
   |   |--B-L_MMM_YYYY_CHANGES.txt 
   |   |--Licence.txt
   |   
   |
   |--DATA
       |--LAYERED
           |--COUNTY_ELECTORAL_DIVISION_REGION
           |--COUNTY_REGION
           |--DISTRICT_BOROUGH_UNITARY_REGION  
           |--DISTRICT_BOROUGH_UNITARY_WARD_REGION
           |--EUROPEAN_REGION_REGION
           |--GREATER_LONDON_CONST_REGION
           |--HIGH_WATER_POLYLINE
           |--PARISH_REGION
           |--SCOTLAND_AND_WALES_CONST_REGION
           |--SCOTLAND_AND_WALES_REGION_REGION
           |--UNITARY_ELECTORAL_DIVISION_REGION
           |--WESTMINSTER_CONST_REGION
       

   -----------
   SECTION TWO   
   -----------

   SECTION TWO - DOC Directory


   Below are the types of documents contained within the DOCUMENTS directory.
     
   o B-L_MMM_YYYY_CHANGES.txt               - Product changes associated with that release.
   o LICENCE.txt                            - contains information relating to your Licence. 
   
 
   The DOCUMENTS directory may contain additional documentation specific to that supply.

   MMM means the month (May, Nov and so on) and YYYY refers to the year.
